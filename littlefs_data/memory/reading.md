# Reading queue

One URL per line, with a `—` and a one-line note on why it's worth reading.
Lango surfaces the **top item** in each morning briefing under 📚 **Today's Read**.

Once you've read something, delete the line (or move it down). Keep this list
to ~10 items max — anything older usually stops being relevant.

## Queue

- https://newsroom.arm.com/news — Arm's own newsroom RSS (use as a backstop; the morning briefing already pulls this)
- https://blogs.windows.com/windowsexperience/ — Microsoft Windows blog (relevant when Windows on Arm news drops)
- https://www.tomshardware.com/tag/chromebooks — Chromebook beat (Snapdragon X Plus / X Elite progress)
- https://www.anandtech.com/tag/cpus — deep-dive CPU coverage when something Arm-related lands
- https://stratechery.com/ — strategy commentary; ARM business model + competitive landscape

## How to add

When you find something you want to read later, paste the URL here with one
line on why. The morning briefing picks the **first** item; rotate or
prioritise by reordering.
